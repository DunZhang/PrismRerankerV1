# Reranker 多级阈值选取方案

## 目标

将 reranker 分数映射为"是正例的概率"，从概率曲线上读取多个阈值，形成分级判断：

| 区间 | 含义 |
|------|------|
| score > T1 | 极大概率是正例（P > 0.9） |
| score > T2 | 较大概率是正例（P > 0.7） |
| score < T3 | 较大概率是负例（P < 0.3） |
| score < T4 | 极大概率是负例（P < 0.1） |

概率档位（0.9/0.7/0.3/0.1）可根据业务需求自行调整。

## 方法：贝叶斯后验概率估计

### 步骤

1. 分别对正例得分和负例得分做 KDE（核密度估计），得到 p(score|正例) 和 p(score|负例)
2. 设定先验概率：P(正例) = 1/8，P(负例) = 7/8（基于 1:7 的正负比）
3. 用贝叶斯公式计算每个分数点的后验概率：P(正例|score) = p(score|正例) × P(正例) / [ p(score|正例) × P(正例) + p(score|负例) × P(负例) ]
4. 从后验概率曲线上，分别找到 P = 0.9 / 0.7 / 0.3 / 0.1 对应的分数值，即为各级阈值
5. 在验证集上计算每个阈值的 Precision 和 Recall，确认效果

### 注意事项

- **先验概率用线上真实比例**：如果线上检索场景的正负比不是 1:7（比如是 1:20），先验要相应调整，这会显著影响阈值位置
- **KDE 的 bandwidth**：默认用 scott 方法即可；样本量小时适当调大，分布复杂时适当调小
- **交叉验证**：建议 70/30 分数据，一份拟合、一份验证，避免过拟合
- **尾部不稳定**：极高分/极低分区域样本稀疏，KDE 估计可能不准，这些区域的阈值需额外人工确认

## 预期产出

一张阈值表，格式如下（数值为示例）：

| 置信档位 | 条件 | 阈值 | Precision | Recall |
|----------|------|------|-----------|--------|
| 极大概率正例 | P(正例) > 0.9 | 0.82 | 0.95 | 0.40 |
| 较大概率正例 | P(正例) > 0.7 | 0.65 | 0.80 | 0.72 |
| 较大概率负例 | P(正例) < 0.3 | 0.35 | — | — |
| 极大概率负例 | P(正例) < 0.1 | 0.18 | — | — |
